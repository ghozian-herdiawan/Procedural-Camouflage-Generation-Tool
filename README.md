# Procedural Camouflage Generation Tool (or just, "PCGT")

To put it simply: It uses math to barf out camouflage patterns. Instead of some poor artist drawing every blotch by hand, we use algorithms to decide where the "paint" goes.

## What is this?

PCGT is a modular suite designed for people who need high-res camo but don't have 77 minutes to wait for a single render. It’s split into two parts so I don't have to break your launcher every time I tweak a math formula.

## Overview

The project is split into two parts to allow for engine updates without requiring a full application re-installation.  

The Launcher
Version 1.0.0: A stable entry point designed to remain unchanged.  

OTA Updates: Automatically pings GitHub to sync the latest pcgt_engine.py core.  

Persistence: Uses version_manifest.json to track local versions and prevent redundant downloads.

The Engine
Version 1.0.2 (or future version): The core logic responsible for rendering.  

Vectorized Processing: Utilizes NumPy to handle high-resolution (4K) exports in seconds rather than hours.  

Pattern Types: Supports Organic, Digital, Honeycomb, and Splinter geometric shapes.  

Randomization: Includes "Randomize All" for new seeds and "Randomize Colors" to swap palettes while keeping current shapes.

## Usage
Launch: Run PCGT_Launcher.exe.  

Sync: The launcher checks for the latest engine file and downloads it if necessary.  

Generate: Adjust sliders for density, scale, and seed to create unique patterns.
